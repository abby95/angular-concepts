What is Http-Interceptor in Angular?
As the name suggests, it is used to intercept outgoing http requests as well as incoming http response.
Few Use Cases:
  • You can add Authorization Token while sending the http request
  • You can format or add your http request
How Have I implemented Http Interceptor in this Code?
  • I created an Http Injectable Class ===> Just like a service
  • Imported Http Interceptor from @angular/common/http
  • You do NOT need a constructor in this file
  • You need a function named intercept() which takes in 2 arguments REQUEST and NEXT HANDLER
  • You can include this in your app.module.ts file under Providers and start using. But there is a better way
THE BETTER WAY

In real world projects, you will create multiple http-interceptors each doing a specific task

  • There could be an Http-Interceptor that removes unwanted data from the http request

  • There could be an Http-Interceptor that Adds Authentication Token to the http request Headers

  • The Better Way is to create all these interceptor files and make a collection of it in a seperate file and export this collection (its nothing but array of all the interceptors)

  • Simply import this provider collections in your app.module.ts file
  • Simply include this provider collections in the Providers Array in your app.module.ts file
What Should you observe?
Check the NETWORKS Tab in your Developers Console. Check the API Call thats being made (in our case its /users)
You will observe that that API Call (HTTP REQUEST) was intercepted by our AuthHeaderInterceptor and it appended/added Auth Token to the Request Header
